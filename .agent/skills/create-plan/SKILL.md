# docs/todo.mdから実装計画を作成する

このスキルは、プロジェクトの `docs/todo.md` に記載された未完了のタスクを整理し、`docs/plans/` ディレクトリに詳細な実装計画書（`.md`）を作成するための手順を定義します。

## 手順

1. **todo.mdの読み取り**
   - `docs/todo.md` の内容を確認し、現在取り組むべきタスクを特定します。特に「- []」で始まっている未完了項目に注目してください。

2. **連番の確認**
   - `docs/plans/` ディレクトリ内の既存ファイルを確認し、次に使用する連番（N）を決定します（例: `5_implementation_plan.md` が最新なら次は `6_...`）。

3. **計画書の作成**
   - `docs/plans/N_plan_name.md` という名前で新しいファイルを作成します。
   - 内容には以下のセクションを含めてください：
     - **概要**: 計画の目的。
     - **タスク一覧**: `todo.md` から抽出した具体的な作業項目。
     - **手動確認チェックリスト**: 実装後に人間が確認すべき項目。

4. **todo.mdの更新（任意）**
   - 必要に応じて `docs/todo.md` に進捗状況（チェックボックスの更新）を反映させます。

## ルール
- 計画書のファイル名は、プロジェクトの既存の規則に従います（例: `N_実装計画.md`）。
- KISS原則に基づき、シンプルなステップで構成してください。
