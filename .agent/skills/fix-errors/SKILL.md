# エラーの修正スキル

このスキルは、プロジェクトのコード品質を保つために `make verify` コマンドを実行し、報告されたエラーや警告を自動的に修正するための手順です。

## 手順

1.  **verifyコマンドの実行**
    *   `make verify` を実行し、コードの検証プロセス（format, lint, build, test）を回します。
    *   コマンドが成功（exit code 0）した場合は、作業終了です。

2.  **エラーの確認**
    *   コマンドが失敗した場合、コンソール出力または以下のログファイルを確認してエラー内容を特定します。
        *   `.tmp/lint.log`: ESLint等の静的解析エラー
        *   `.tmp/build.log`: ビルドエラー、TypeScriptの型エラー
        *   `.tmp/test.log`: テストの失敗

3.  **エラーの分析と修正**
    *   ログに記載されたファイルパス、行数、エラーメッセージを読み解きます。
    *   エラーの種類に応じて適切な修正を行います（例: 未使用変数の削除、型定義の修正、フックの依存配列修正など）。
    *   **注意**: 修正は慎重に行い、既存の機能を破壊しないようにしてください。

4.  **再検証**
    *   修正後、再度 `make verify` を実行します。
    *   全てのエラーが解消されるまで手順2と3を繰り返します。

## ルール

*   **自動修正の優先**: 可能な場合、`npm run lint --fix` などの自動修正ツールを活用しますが、手動修正が必要なことが多い点に注意してください。
*   **ログの活用**: 詳細なエラー内容は必ずログファイル（`.tmp/` 配下）を参照してください。
*   **副作用の回避**: エラーを黙らせる（`eslint-disable` 等）のではなく、可能な限り根本的な解決を目指してください。
