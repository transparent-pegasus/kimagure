# 変更内容のGit同期

このスキルは、現在の作業による変更内容をGitリポジトリに同期（コミットおよびプッシュ）するための手順を定義します。

## 手順

1. **変更内容の確認**
   - `git status` および `git diff` を使用して、コミット対象のファイルと具体的な修正内容を確認します。

2. **ステージング**
   - `git add .` または個別のファイルパスを指定して、変更をステージングエリアに追加します。

3. **コミットメッセージの作成**
   - 変更内容に基づき、簡潔で分かりやすいコミットメッセージを作成します。
   - 形式例: `feat: add google login`, `fix: navigation issue`

4. **コミットとプッシュ**
   - `git commit -m "メッセージ"` を実行します。
   - `git push` を実行してリモートリポジトリに反映させます（リモートが設定されている場合）。

## 注意事項
- `GEMINI.md` に「Git操作はユーザーが管理する」という制限があるため、このスキルはユーザーから明示的な指示（「コミットして」「プッシュして」等）があった場合にのみ使用してください。
- `make verify` が通過していることを事前に確認することを推奨します。
