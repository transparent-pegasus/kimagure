# 作業ディレクトリの変更管理

このスキルは、プロジェクト内の特定のサブディレクトリ（例: `frontend/`, `backend/`）で作業を行う際のディレクトリ管理手順を定義します。

## 手順

1. **ターゲットディレクトリの特定**
   - 作業対象がどのサブディレクトリにあるかを確認します（例: Frontendなら `frontend/`）。

2. **コマンド実行時のCWD指定**
   - `run_command` ツールを使用する際、必ず `Cwd` 引数に対象の絶対パスを指定します。
   - 例: `Cwd: "c:/Users/kunim/Documents/repos/kimagure/frontend"`

3. **コンテキストの維持**
   - 以降のタスクにおいて、そのディレクトリが「カレント」であると想定してツール（`list_dir`, `grep_search` 等）を使用します。

## ルール
- シェル自体の `cd` コマンドは永続しないため、ツール呼び出しごとに `Cwd` を明示することを徹底します。
- ユーザーにディレクトリ変更を報告する場合、プロジェクトルートからの相対パスで伝えると親切です。
