# [実装計画] UI/UX改善と仕様追加

## 概要
`docs/todo.md` に基づき、UI/UXの改善、デフォルト値の変更、および仕様の追記を行う。

## ユーザーレビューが必要な事項(確認済)
- **提案結果画面の配色**: カロリー・栄養素表の配色案（ピンク等の柔らかい色）を実装時に提示し、決定する。

## 変更内容

### ドキュメント更新 (`AGENTS.md`)
- **[仕様追記]** 「記録(Log)」において具体的な商品名が入力された場合、その栄養素を検索して埋める旨を追記。
- **[入力欄UI]** 商品名入力が有効であることをプレースホルダー等で明示する方針を追記。

### フロントエンド実装 (`frontend/app/page.tsx`, `components/`)

#### 1. ヘッダー修正
- タイトルを「きまぐれごはん」に変更。
- サブタイトルを `計画しないがいちばん続く` に変更（括弧・句点削除）。

#### 2. 入力フォーム改善
- **テキスト**: 「食べたもの・予定を入力...」の文字サイズを拡大。
- **プレースホルダー**: 具体的な商品名も有効であることを示す文言に変更。
- **デフォルト値**:
  - 食事: 全て空（朝・昼・夜）、提案対象は未選択状態。
  - 手間: 「買う」。
  - バランス: 「日単位」。
  - NG食材: デフォルトなし。
- **食事区分**: 昼と夜の間に「おやつ」に名称変更。

#### 3. 条件設定・NG食材
- **スイッチ**: 操作不能を修正する。配色を修正（暖色/寒色で中立的な色へ）。
- **NG食材**:
  - 「0g」かつ「数値入力ロック」の「NG」スイッチを追加。
  - 数値入力欄の二重ボーダーを解消。
- **ボタン**: 「制限項目を追加」のクリッカビリティ向上（見た目の調整）。
- **レイアウト**: 最下部スクロール時、「制限項目を追加」の下に十分な余白を確保。

#### 4. 提案（Target）表示
- 「提案対象(選択中)」の表示をより目立たせる（色味、強調など）。

#### 5. 提案結果画面 (`currentScreen === "result"`)
- **タイトル**: 「ご提案メニュー」→「今日の[夜ごはん]」（対象枠に応じて可変に）。
- **チャート配色**: 黒ベースから柔らかい色（ピンク系など）へ変更。
- **決定ボタン**: 赤系の暖色に変更。

## 検証計画

### 自動テスト
- `make verify` によるビルド・Lintチェック。

### 手動確認
- デフォルト値が正しいか。
- 「夕方」の代わりに「おやつ」が追加されるか。
- ヘッダー文言の確認。
- NG食材の0gロック挙動。
- 提案結果画面の配色とタイトルの可変性。
