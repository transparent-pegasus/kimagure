# 実装計画: バグ修正とUI改善

## 概要
`docs/todo.md` に記載された以下の課題に対応するための実装計画です。
1. メニュー遷移時の不具合修正
2. 週単位調整機能の一時無効化

## タスク一覧

### 1. メニュー遷移時の不具合修正
- **現状**: メニューから「プライバシーポリシー」などのページに遷移した後、メニューが開いたままになっている。
- **目標**: ページ遷移時にメニューが自動的に閉じるようにする。
- **実装方針**:
  - `Header` コンポーネントまたは `Menu` コンポーネント内のリンククリックイベントハンドラを修正し、遷移時に `isOpen` 状態を `false` に更新する処理を追加する。

### 2. 週単位調整機能の一時無効化
- **現状**: 「週単位の調整」機能は過去6日分のPFC数値を参照する必要があるが、バックエンドAPIが未実装の場合でも選択可能になっている可能性がある。
- **目標**: バックエンド側の対応が完了するまで、この機能を一時的に選択不可（disabled）にするか、UI上で非表示にする。
- **実装方針**:
  - 「週単位の調整」を選択するUI要素（ラジオボタンやセレクトボックス等）に対し、`disabled` 属性を付与するか、選択肢から除外する。
  - ユーザーに対して「準備中」などのメッセージを表示することを検討する。

## 手順

1. `Header.tsx` (または関連コンポーネント) を調査・修正し、ナビゲーションリンクのクリック時にメニューを閉じる処理を追加。
2. 「週単位の調整」機能に関連するコンポーネントを特定し、入力不可にする制御を追加。
3. 動作確認。

## 手動確認チェックリスト
- [ ] メニューから「プライバシーポリシー」をクリックした際、ページ遷移と同時にメニューが閉じること。
- [ ] 他のメニュー項目からの遷移でも同様にメニューが閉じること。
- [ ] 「条件設定」画面等で「週単位の調整」が選択できない（グレーアウトされている、または選択肢にない）こと。
